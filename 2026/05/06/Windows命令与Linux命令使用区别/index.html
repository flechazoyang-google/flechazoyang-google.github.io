Windows命令与Linux命令使用区别

一、终端环境差异

对比项WindowsLinux
默认 ShellCMD / PowerShellBash / Zsh
大小写敏感❌ 不敏感✅ 敏感
路径分隔符\（反斜杠）/（正斜杠）
盘符概念有（C:、D:）无（统一根目录 /）
脚本扩展名.bat / .ps1无固定扩展名

注：PowerShell 兼容部分 Linux 命令别名（如 ls），但底层实现不同。

九、常见坑点

路径分隔符混用

  • Windows 的 CMD 接受 \，但也兼容 /
  • 但在 PowerShell 中，\ 是转义符，路径建议用 /
  • Linux 只认 /

通配符差异

空格处理

换行符

  • Windows：CRLF（\r\n）
  • Linux：LF（\n）
  • 跨平台脚本注意用 git config --global core.autocrlf true 自动转换
